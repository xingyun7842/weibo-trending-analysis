# Weibo Trending Product Ideas Analyzer

一个基于Claude Code的自动化微博热搜分析和产品创意生成工具。

## 功能概述

这个skill可以帮助你：

1. **自动抓取微博热搜榜单** - 通过API获取实时热搜话题
2. **深度研究热点背景** - 使用Web搜索工具自动收集相关新闻和背景信息
3. **智能分析评分** - 基于有趣度（80分）+ 有用度（20分）的模型评估每个热点
4. **生成产品创意** - 为每个热点提取可行的产品创意方案
5. **导出HTML报告** - 生成精美的可视化分析报告

## 快速开始

### 方式一：直接运行（推荐）

**已预配置TianAPI接口**，无需额外配置，直接运行：

```bash
/weibo-trending-product-ideas
```

Skill会自动：
1. 从TianAPI获取最新微博热搜（已内置API密钥）
2. 分析前10-15个热点话题
3. 使用WebSearch搜索每个话题的详细信息
4. 生成产品创意并进行评分
5. 输出精美的HTML分析报告

**预计用时**: 5-10分钟

### 方式二：自定义配置

如果你有自己的TianAPI账号或其他API服务：

1. 编辑 `config.json` 文件
2. 修改API密钥或端点
3. 运行 `/weibo-trending-product-ideas`

## 评分标准

### 有趣度 (80分)
- **病毒传播潜力** (0-20分): 话题的传播性和扩散能力
- **情感吸引力** (0-20分): 能否引发情感共鸣
- **社区参与度** (0-20分): 用户讨论和互动的热度
- **文化相关性** (0-20分): 与当前文化趋势的契合度

### 有用度 (20分)
- **问题解决潜力** (0-10分): 能否解决实际问题
- **市场需求** (0-10分): 市场需求的强度和规模

### 评级分类
- **优秀** (≥80分): 强烈推荐，高潜力产品创意
- **良好** (60-79分): 值得考虑的产品创意
- **一般** (<60分): 参考价值有限

## 产品创意格式

每个产品创意包含：

- **产品名称**: 创意、易记的中文名称
- **核心功能**: 3-5个关键功能点
- **目标用户**: 详细的用户画像（人口统计、需求、行为特征）
- **创新角度**: 独特价值主张和创新点
- **评分明细**: 有趣度、有用度和综合得分

## HTML报告特点

生成的HTML报告包含：

- **响应式设计**: 适配桌面和移动设备
- **分级展示**: 根据评分高亮优秀和良好创意
- **事件时间线**: 展示热点事件的发展脉络
- **视觉化评分**: 直观的评分展示和对比
- **完整引用**: 包含信息来源链接

## 技术架构

### 数据流程

```
1. 数据获取
   ├─ 使用Bash + curl调用TianAPI
   └─ 解析JSON响应，提取热搜列表

2. 信息研究
   ├─ 对每个热点使用WebSearch搜索新闻
   ├─ 收集3-5个相关信息源
   └─ 构建事件时间线和背景分析

3. AI分析
   ├─ 评估有趣度 (80分制)
   │  ├─ 病毒传播潜力 (0-20)
   │  ├─ 情感吸引力 (0-20)
   │  ├─ 社区参与度 (0-20)
   │  └─ 文化相关性 (0-20)
   ├─ 评估有用度 (20分制)
   │  ├─ 问题解决潜力 (0-10)
   │  └─ 市场需求 (0-10)
   └─ 生成1-3个产品创意

4. 报告生成
   ├─ 按评分排序所有创意
   ├─ 应用分级样式 (优秀/良好/一般)
   ├─ 生成HTML文件
   └─ 输出统计摘要
```

### 配置管理

Skill支持通过 `config.json` 灵活配置：

```json
{
  "api": {
    "tianapi": {
      "endpoint": "API端点",
      "key": "API密钥"
    }
  },
  "analysis": {
    "maxTopicsToAnalyze": 15,        // 分析热点数量
    "minScoreForExcellent": 80,      // 优秀标准
    "minScoreForGood": 60,           // 良好标准
    "scoreWeights": {
      "interest": 80,                // 有趣度权重
      "utility": 20                  // 有用度权重
    }
  },
  "report": {
    "outputDirectory": ".",          // 报告输出目录
    "fileNameTemplate": "...",       // 文件名模板
    "language": "zh-CN"              // 报告语言
  }
}
```

### 工具使用

- **Bash**: 执行curl命令获取API数据
- **WebSearch**: 搜索热点相关新闻和背景（每个热点3-5次搜索）
- **Write**: 生成HTML报告文件
- **AI分析**: Claude进行创意生成和评分

## 注意事项

1. **API限制**: 确保你的API有足够的调用配额
2. **网络要求**: 需要稳定的网络连接进行Web搜索
3. **处理时间**: 分析多个热点可能需要几分钟时间
4. **数据时效性**: 建议定期运行以获取最新热点

## 配置权限

这个skill需要以下权限：
- `WebSearch`: 搜索热点相关信息

## 常见问题

### Q: 如何开始使用？
A: 直接运行 `/weibo-trending-product-ideas`，已预配置TianAPI接口，无需额外设置。

### Q: API接口格式不匹配怎么办？
A: 编辑 `prompt.md` 中Step 1的API解析逻辑，适配你的API响应格式。

### Q: 如何修改分析数量？
A: 编辑 `config.json`，修改 `analysis.maxTopicsToAnalyze` 的值（建议10-20之间）。

### Q: 如何调整评分标准？
A: 编辑 `config.json`：
- `minScoreForExcellent`: 修改优秀标准（默认80）
- `minScoreForGood`: 修改良好标准（默认60）
- `scoreWeights`: 修改有趣度和有用度的权重比例

### Q: 如何更换API密钥？
A: 编辑 `config.json`，修改 `api.tianapi.key` 为你的密钥。

### Q: 可以分析其他平台的热搜吗？
A: 可以，需要：
1. 获取对应平台的API接口
2. 修改 `config.json` 中的API配置
3. 调整 `prompt.md` 中的数据解析逻辑

### Q: 生成的HTML报告在哪里？
A: 默认保存在当前工作目录，文件名格式为 `weibo_trending_analysis_report_{时间戳}.html`。可在 `config.json` 中修改输出目录。

### Q: 分析需要多长时间？
A: 分析15个热点约需5-10分钟，每个话题需要进行Web搜索和AI分析。

### Q: WebSearch搜索失败怎么办？
A: Skill会自动跳过无法搜索的话题，继续分析下一个，并在报告中标注。

### Q: 如何查看样式模板？
A: 打开 `template.html` 文件可以看到完整的HTML报告设计和样式。

### Q: 如何测试API是否正常？
A: 运行以下命令测试：
```bash
curl -s "https://apis.tianapi.com/weibohot/index?key=你的密钥" | head -50
```
返回 `"code": 200` 表示正常。
