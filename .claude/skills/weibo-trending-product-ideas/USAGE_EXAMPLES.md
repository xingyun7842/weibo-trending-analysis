# 使用示例与测试指南

## 快速开始

### 1. 基本使用

最简单的使用方法:

```bash
/weibo-trending-product-ideas
```

执行后,Skill会自动:
- 从TianAPI获取最新微博热搜
- 分析前10-15个热点话题
- 生成产品创意和评分
- 输出HTML分析报告

## 自定义配置

### 修改分析数量

编辑 `config.json`:

```json
{
  "analysis": {
    "maxTopicsToAnalyze": 20,  // 改为分析前20个热点
    ...
  }
}
```

### 修改评分标准

编辑 `config.json`:

```json
{
  "analysis": {
    "minScoreForExcellent": 85,  // 改为85分以上才是优秀
    "minScoreForGood": 70,       // 改为70分以上才是良好
    ...
  }
}
```

### 更换API密钥

如果你有自己的TianAPI账号,编辑 `config.json`:

```json
{
  "api": {
    "tianapi": {
      "key": "你的API密钥",
      ...
    }
  }
}
```

## 常见问题排查

### Q1: API调用失败

**错误信息**: "Unable to fetch from apis.tianapi.com"

**解决方案**:
1. 检查网络连接
2. 验证API密钥是否有效
3. 确认API配额未用尽
4. Skill会自动使用Bash+curl作为备用方案

### Q2: WebSearch返回空结果

**可能原因**:
- 话题太新,搜索引擎尚未收录
- 搜索关键词需要优化

**Skill会自动**:
- 记录无法搜索的话题
- 继续分析下一个话题
- 在报告中标注信息来源缺失

### Q3: 生成HTML报告失败

**检查事项**:
1. 当前目录是否有写入权限
2. 磁盘空间是否充足
3. 文件名是否包含非法字符

### Q4: 分析速度慢

**正常情况**:
- 分析15个热点约需5-10分钟
- 每个话题需要进行Web搜索和AI分析

**优化建议**:
- 减少 `maxTopicsToAnalyze` 数量
- 使用更快的网络连接

## 高级用法

### 仅分析特定类型的热点

虽然Skill会分析所有热点,但你可以在执行后手动筛选:

1. 打开生成的HTML报告
2. 使用浏览器的查找功能 (Ctrl+F / Cmd+F)
3. 搜索特定关键词(如"科技"、"娱乐"等)

### 批量分析历史数据

如果你保存了多天的分析报告,可以:

1. 比较不同日期的热点趋势
2. 追踪特定话题的演变
3. 识别长期市场机会

HTML报告可以:
- 在浏览器中打开

- 复制内容到文档
- 分享给团队成员
